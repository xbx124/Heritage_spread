
# 非遗文化数字化传播系统 — 后端 API 接口文档（v2.0 完整版）

> **本文档由组长最终定义，成员B按此实现，成员C/D严格按此调用，定稿后不再随意修改**  
> 基础地址：`http://localhost:8080/api`  
> 统一字符编码：UTF-8  
> 数据格式：JSON  
> 认证方式：JWT Bearer Token（除登录、注册、公开查询接口外，均需在 `Authorization` 头携带 `Bearer <token>`）

---

## 一、通用约定

### 1.1 统一响应结构
```json
{
  "code": 200,
  "msg": "success",
  "data": {}
}
```

| code | 含义             |
|------|------------------|
| 200  | 请求成功          |
| 400  | 参数错误/业务规则错误 |
| 401  | 未认证或 token 失效 |
| 403  | 权限不足          |
| 404  | 资源不存在        |
| 500  | 服务器内部错误    |

> `msg` 字段用于描述具体错误原因，如“用户名已存在”、“数据不存在”等。

### 1.2 分页响应格式
对于分页列表接口，`data` 字段包含：
```json
{
  "total": 50,
  "page": 1,
  "size": 10,
  "list": [ ... ]
}
```

### 1.3 认证说明
- 登录成功后返回 `token`，前端存储于 `localStorage`。
- 每次请求需在请求头添加：`Authorization: Bearer <token>`。
- `token` 有效期设定为 **7 天**，过期后需重新登录。

## 二、认证接口

### 2.1 用户注册（可选，演示阶段可仅后台录入）

```
POST /api/auth/register
```

**权限**：公开  
**请求体**：
```json
{
  "username": "user2",
  "password": "123456",
  "role": "user"          // 可选，默认为 "user"，管理员注册由后台控制
}
```

**成功响应**：
```json
{
  "code": 200,
  "msg": "注册成功",
  "data": {
    "user_id": 3,
    "username": "user2",
    "role": "user"
  }
}
```

### 2.2 用户登录

```
POST /api/auth/login
```

**权限**：公开  
**请求体**：
```json
{
  "username": "editor1",
  "password": "123456"
}
```

**成功响应**：
```json
{
  "code": 200,
  "msg": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "user_id": 2,
      "username": "editor1",
      "role": "editor"
    }
  }
}
```

### 2.3 获取当前用户信息

```
GET /api/auth/me
```

**权限**：登录用户  
**请求头**：`Authorization: Bearer <token>`  
**成功响应**：
```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "user_id": 2,
    "username": "editor1",
    "role": "editor",
    "create_time": "2025-04-01 10:00:00"
  }
}
```

## 三、非遗分类接口

### 3.1 获取所有分类（公开）

```
GET /api/categories
```

**权限**：无  
**返回示例**：
```json
{
  "code": 200,
  "msg": "success",
  "data": [
    { "category_id": 1, "category_name": "传统技艺", "category_desc": "包括各类手工技艺..." },
    { "category_id": 2, "category_name": "传统戏剧", "category_desc": "包括地方戏曲..." }
  ]
}
```

### 3.2 新增分类（管理员）

```
POST /api/categories
```

**权限**：`admin`  
**请求头**：`Authorization: Bearer <admin_token>`  
**请求体**：
```json
{
  "category_name": "传统音乐",
  "category_desc": "包含古琴、二胡等传统乐器及音乐形式"
}
```

**成功响应**：
```json
{
  "code": 200,
  "msg": "分类创建成功",
  "data": { "category_id": 5 }
}
```

### 3.3 编辑分类（管理员）

```
PUT /api/categories/:id
```

**权限**：`admin`  
**请求体**：
```json
{
  "category_name": "传统音乐（修订）",
  "category_desc": "补充说明..."
}
```

### 3.4 删除分类（管理员）

```
DELETE /api/categories/:id
```

**权限**：`admin`  
**成功响应**：`{ "code":200, "msg":"删除成功" }`  
*注意：若分类下有非遗项目，则不允许删除，返回 400 并给出提示。*

| page        | int | 否 | 页码，默认 1 |
| size        | int | 否 | 每页条数，默认 10 |

```json
{
  "code": 200,

  "data": {

    "size": 10,

        "inheritor_count": 3,
        "create_time": "2025-05-01 10:00:00"
      }
    ]
  }
}
```

### 4.2 获取项目详情（公开）

```
GET /api/projects/:id
```

### 4.3 新增项目（管理员/编辑）

```
POST /api/projects
```

**权限**：`admin` / `editor`  
**请求头**：`Authorization: Bearer <token>`  
**请求体**：
```json
{
  "project_name": "京剧",
  "category_id": 2,
  "area": "北京",
  "project_intro": "京剧又称为平剧、京戏...",
```

```json
{
  "code": 200,
  "msg": "项目创建成功",
  "data": { "project_id": 6 }
}
```

```
PUT /api/projects/:id
```

**权限**：`admin` / `editor`  
**请求体**：与新增类似，可部分更新。若传递 `inheritors` 则全量替换关联的传承人（可选）。

```
DELETE /api/projects/:id
```

**权限**：`admin`  
**功能**：逻辑下架（设置 `status=0`）或物理删除，防止普通用户删除操作。

### 6.2 获取资讯详情（公开）

```
GET /api/news/:id
```

```json
{
  "code": 200,
  "msg": "success",
  "data": {

    "content": "正文内容（可能包含HTML）...",
```

```
POST /api/news
```

```json
{
  "title": "新活动预告",
  "content": "<p>正文HTML内容</p>",
  "cover_image": "https://.../cover.jpg",   // 可选

```

**成功响应**：
```json
{
  "code": 200,
  "msg": "资讯发布成功",
  "data": { "news_id": 21 }
}
```

```
PUT /api/news/:id
```

**权限**：`editor`（仅限自己发布的资讯）或 `admin`

```
DELETE /api/news/:id
```

**权限**：`editor`（仅限自己）或 `admin`  
**功能**：逻辑删除，已发布的资讯不可物理删除，管理员可下架。

### 6.6 审核资讯（管理员）

```
PUT /api/news/:id/audit
```

**权限**：`admin`  
**请求体**：
```json
{
  "status": "published"   // published / rejected
}
```

## 基础约定

- **请求格式**：所有接口统一使用 JSON 格式，请求头需添加 `Content-Type: application/json`。
- **响应格式**：统一返回以下结构（无论成功或失败）：

```json
{
  "code": 200,
  "msg": "success",
  "data": { }
}
```

| 状态码 | 含义         | 说明               |
|--------|--------------|-------------------|
| 200    | 成功         | 请求处理成功       |
| 400    | 参数错误     | 客户端参数不合法   |
| 401    | 未授权       | 缺少或无效 Token  |
| 403    | 无权限       | 角色权限不足       |
| 404    | 数据不存在   | 请求的资源未找到   |
| 500    | 服务器错误   | 内部异常           |

- **字段命名**：所有字段使用**下划线命名法**（如 `project_name`，不用驼峰）。
- **时间格式**：统一使用 `YYYY-MM-DD HH:mm:ss`（如 `"2025-05-01 10:00:00"`）。

---

## 鉴权说明

系统采用 **JWT（JSON Web Token）** 进行身份验证。

- 登录成功后，后端返回 Token，前端需保存在 `localStorage`。
- 除公开接口（标注为“✅ 公开”）外，其余接口均需在请求头中携带：

```
Authorization: Bearer <token>
```

- 各接口**权限**标注含义：  
  - `PUBLIC`：无需登录  
  - `USER`：任意登录用户（`user`/`editor`/`admin`）  
  - `EDITOR`：仅 `editor` 或 `admin`  
  - `ADMIN`：仅 `admin`

## 一、用户认证

### 1.1 登录（PUBLIC）

```
POST /api/auth/login
```

**请求体**

| 参数     | 类型   | 必填 | 说明     |
|----------|--------|------|----------|
| username | string | 是   | 登录账号 |
| password | string | 是   | 明文密码 |

**成功响应**

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "user_id": 2,
      "username": "editor1",
      "role": "editor"
    }
  }
}
```

### 1.2 获取当前用户信息（USER）

```
GET /api/auth/me
```

**成功响应**（返回 Token 对应用户的完整信息）

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "user_id": 2,
    "username": "editor1",
    "role": "editor",
    "create_time": "2025-05-01 09:00:00"
  }
}
```

> **说明**：前端通过该接口校验 Token 有效性，并获取角色信息。

## 四、非遗项目

### 4.1 获取项目列表（PUBLIC）

```
GET /api/projects
```

**请求参数（可选）**

| 参数        | 类型   | 必填 | 说明                      |
|-------------|--------|------|---------------------------|
| category_id | int    | 否   | 按分类筛选                |
| area        | string | 否   | 按地区筛选                |
| page        | int    | 否   | 页码，默认 1             |
| size        | int    | 否   | 每页条数，默认 10，最大 50|

**分页响应示例**

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "total": 25,
    "list": [
      {
        "project_id": 1,
        "project_name": "景德镇手工制瓷技艺",
        "category_name": "传统技艺",
        "area": "江西景德镇",
        "create_time": "2025-05-01 10:00:00"
      }
    ]
  }
}
```

> **注意**：`data` 为分页对象，包含 `total`（总数）和 `list`（当前页数据）。当查询无结果时，返回 `{"total":0,"list":[]}`。

### 4.2 获取项目详情（PUBLIC）

```
GET /api/projects/:id
```

**返回示例**

```json
      {
        "inheritor_id": 1,
        "name": "王龙根",
```

```
GET /api/news
```

**请求参数**

| 参数 | 类型 | 必填 | 说明                |
|------|------|------|---------------------|

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "total": 34,
    "list": [
      {
        "news_id": 1,
        "title": "昆曲进校园活动正式启动",
        "author": "editor1",
        "publish_time": "2025-05-10 09:00:00"
      }
    ]
  }
}
```

### 6.2 获取资讯详情（PUBLIC）

```
GET /api/news/:id
```

**返回示例**

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "news_id": 1,
    "title": "昆曲进校园活动正式启动",
    "content": "正文内容...（支持 HTML 或纯文本，后端需保证 XSS 过滤）",
    "author": "editor1",
    "publish_time": "2025-05-10 09:00:00"
  }
}
```

### 6.3 发布资讯（EDITOR）

```
POST /api/news
```

**请求头**：`Authorization: Bearer <token>`

**请求体**

| 参数    | 类型   | 必填 | 说明       |
|---------|--------|------|------------|
| title   | string | 是   | 资讯标题   |
| content | string | 是   | 资讯正文   |

**成功响应**

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "news_id": 35
  }
}
```

### 6.4 编辑资讯（EDITOR）

```
PUT /api/news/:id
```

**请求头**：`Authorization: Bearer <token>`

**请求体**（只传需要修改的字段）

| 参数    | 类型   | 必填 | 说明       |
|---------|--------|------|------------|
| title   | string | 否   | 新标题     |
| content | string | 否   | 新正文     |

**权限说明**：普通 `editor` 只能修改自己发布的资讯；`admin` 可修改所有资讯。

**成功响应**

```json
{
  "code": 200,
  "msg": "success",
  "data": null
}
```

### 6.5 删除资讯（EDITOR）

```
DELETE /api/news/:id
```

**权限说明**：同编辑接口。

**成功响应**

```json
{
  "code": 200,
  "msg": "success",
  "data": null
}
```

## 七、传播渠道管理

### 7.1 获取渠道列表（PUBLIC / ADMIN）

```
GET /api/channels
```

**返回示例**

```json
{
  "code": 200,
  "msg": "success",
  "data": [
    {
      "channel_id": 1,
      "channel_name": "官方网站",
      "channel_type": "线上-网站"
    },
    {
      "channel_id": 2,
      "channel_name": "抖音短视频",
      "channel_type": "线上-短视频"
    }
  ]
}
```

> 公开接口可直接用于前端展示，管理操作需 admin 权限。

### 7.2 新增渠道（ADMIN）

```
POST /api/channels
```

**请求体**

| 参数         | 类型   | 必填 | 说明                 |
|--------------|--------|------|----------------------|
| channel_name | string | 是   | 渠道名称             |
| channel_type | string | 是   | 渠道类型（如"线上-网站"） |

**成功响应**

```json
{
  "code": 200,
  "msg": "success",
  "data": { "channel_id": 5 }
}
```

### 7.3 编辑渠道（ADMIN）

```
PUT /api/channels/:id
```

**请求体**（至少一个字段）

| 参数         | 类型   | 必填 | 说明     |
|--------------|--------|------|----------|
| channel_name | string | 否   | 新名称   |
| channel_type | string | 否   | 新类型   |

**成功响应**

```json
{
  "code": 200,
  "msg": "success",
  "data": null
}
```

### 7.4 删除渠道（ADMIN）

```
DELETE /api/channels/:id
```

**成功响应**

```json
{
  "code": 200,
  "msg": "success",
  "data": null
}
```

> **注意**：删除渠道前应检查是否有关联传播数据，可设定外键 `ON DELETE SET NULL` 或禁止删除。
